CTR

CTR manages the CTR list. Click the CTR in the left navigation menu under SAR & CTR, a new CTR Tab folder will appear in the right panel. There are 3 sub-Tab folders: Search CTR, Search result CTR and Create New CTR.

  • Search CTR
  • Search Result CTR
  • Update/Delete CTR
  • Create New CTR

a) Search CTR

Search Criteria

CTR Id This is the system generated unique number for CTR entry in the system.
CTR Title Name of CTR, used to refer to a CTR.
DCN# Document Control number. It is optional
BSA Id Internal BSA Id, it is optional.
Org unit Code The org unit this CTR belongs to. It is used to control access
Assign To Assign to an individual
Each CTR has a status and can assign to an individual or a group.
Assign To Group Assign to a Group
Status Code Current status of the CTR
Assign Date The date when the CTR is assigned to
AND and OR Select AND radio box if you want those criteria add together.
Select OR radio box if you want to get the result of either criteria meet

Functional buttons

  1. Find:
    click Find button to search.
  2. Clear:
    click Clear button to clear all the criteria.

b) Search Result CTR

The top part is the table which shows the list of the CTRs that meet the search criteria. The bottom part are the details. There are Details, Person Involved in Txn(4A), Txn Took Place FI(2A), Document, Email and Action Log Tabs,

In the Details Tab, there are CTR - General, Type of Filing, Parent FI(2A) and Transaction Summary(3A) sub Tabs
Each CTR can have 1 to many Person Involved in Txn(4A) and Txn Took Place FI(2A) records.

c) Update CTR

Double Click the selected record in the table to modify this entry. A new Tab folder "Update CTR" will be added, as show below.

Action Options

  1. CTR Action:
    Select an action to perform. The action allowed to perform is configured via the workflow along with the user permission.
  2. Action Reason:
    Select a pre-defined action reason in the dropdown menu.
  3. Comments:
    Add comments in the comments text area.

Functional buttons

  1. Submit:
    Depends on the action the user selected. Some actions will open a pop-up window or open a new Tab to allow the user to input more data.
  2. Reset:
    reset the data to its initial value, discard all the changes.
  3. Close:
    Discard this change (if any) and switch back to "Search result CTR" Tab.

d) Create New CTR:

Select the "Create New CTR" Tab to create a new entry. The mandatory fields in all tabs must be filled.
The easy way to create a new CTR is by copying from another One.

Field Name Descriptions.
CTR Id This is the system generated unique number for CTR entry in the system.

Click on the copy icon to open a new wizard window to find a CTR to copy from.
CTR Title Name of CTR, used to refer to a CTR.
Case Id The CTR is linked to the case Id if the SAR is created from a Case in the Case Management system.
DCN# Document Control number. It is optional
BSA Id Internal BSA Id, it is optional.
Assign To Assign to an individual
Each CTR has a status and can assign to an individual or a group.
Assign To Group Assign to a Group
Org unit Code The org unit this CTR belongs to. It is used to control access
CTR Explain Explain why we need to create this CTR
Action Reason Select a pre-defined action reason in the dropdown menu.
Comments Add comments on creating the CTR

Functional buttons

  1. Create:
    Click the Create button to create a new CTR and switch to "Search result CTR" Tab. Some fields are mandatory, an error pop-up window will show if the data does not pass the validation.
  2. Reset:
    Click the Reset button to reset all the fields to initial value.
  3. Close:
    Click the Close button to cancel operation and switch to "Search result CTR" Tab.
